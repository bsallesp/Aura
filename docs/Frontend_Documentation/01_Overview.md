# Overview

## Purpose
The **Aura** frontend application is a modern, responsive single-page application (SPA) built with **Angular**. Its primary purpose is to provide a seamless interface for aesthetic clinics to manage their operations and for clients to book appointments. It serves as the client-side interface for the Aesthetic Appointment Management System, interacting with the .NET backend API.

## Key Features
- **Appointment Booking**: Interactive calendar and booking flow for clients.
- **Professional Dashboard**: Schedule management, availability configuration, and appointment oversight.
- **Service Management**: CRUD operations for aesthetic services (pricing, duration, policies).
- **Financial Integration**: Payment processing via Stripe elements and financial reporting.
- **Authentication**: Secure login/registration for Professionals, Admins, and Clients using JWT.
- **Real-time Updates**: Status changes and notifications.

## User Personas
1.  **Client**:
    - Browses services and professionals.
    - Books and manages appointments.
    - Views history and makes payments.
2.  **Professional**:
    - Manages personal availability and schedule.
    - Views upcoming appointments and client details.
    - Configures service offerings and policies.
3.  **Administrator**:
    - Oversees platform usage.
    - Manages tenant configurations and high-level settings.
    - Accesses audit logs and system health metrics.
